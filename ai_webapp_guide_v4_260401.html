여러 팀이 하나의 스프레드시트를 공유하며 데이터를 입력하고 관리할 때, 아래와 같은 불편함을 겪고 있다면 이 가이드가 도움이 됩니다.

❌ 공유 시트 방식의 한계
  • 여러 명이 동시에 수정 → 데이터 충돌
  • 실수로 수식이나 기존 데이터 삭제
  • 누가 언제 수정했는지 추적 어려움

  • 잘못된 입력 자동 차단 (검증)
  • PC·모바일 모두 사용 가능
  • 업무 흐름에 맞게 UI 맞춤 설정

어떤 팀이든 만들 수 있는 웹앱 유형

📊 현황 대시보드 팀의 주요 수치를 차트와 통계로 보여주는 화면
📝 데이터 입력 폼 팀원이 직접 데이터를 입력하고 저장하는 화면
🔍 조회 및 검색 도구 기존 데이터를 필터·검색해서 확인하는 화면
✅ 검수·확인 도구 두 데이터를 비교하거나 누락·오류를 찾아주는 화면
팀별 적용 예시
  • 영업팀: 거래처별 실적 입력 및 월별 달성률 대시보드
  • 생산팀: 일별 생산량 입력 및 불량률 차트
  • 구매팀: 발주 현황 입력 및 납기 관리 조회
  • 인사팀: 교육 이수 현황 입력 및 완료율 현황판
  • 품질팀: 검사 결과 입력 및 불일치 항목 자동 표시

사용하는 두 가지 도구

ChatGPT

웹앱의 코드를 대신 작성해주는 AI. 내 업무를 설명하면 HTML 파일 형태의 코드를 만들어줍니다. 코딩을 전혀 몰라도 됩니다.

Claude Cowork

ChatGPT가 만든 코드를 실행해서 실제 웹앱처럼 볼 수 있게 해주는 도구. 수정 요청도 말로 하면 됩니다.

⚙️

Claude Cowork란?

본격적으로 시작하기 전, 핵심 개념을 이해합니다

[File] 파일 직접 접근

지정 폴더의 파일을 직접 읽고, 편집하고, 새로 생성합니다. 별도 업로드 없이 로컬 파일에 바로 접근합니다.

[Auto] 멀티스텝 자동화

복잡한 작업을 단계별로 나눠 순서대로 자율 실행합니다. 중간에 사람이 개입하지 않아도 됩니다.

[Done] 완성된 결과물

HTML 웹앱, Excel, PPT 등 바로 사용할 수 있는 완성본을 파일로 저장해줍니다.

일반 Chat과의 차이

일반 Chat은 답변만 줍니다. Cowork는 실제로 파일을 만들고, 코드를 실행하고, 결과물을 저장하는 행동하는 AI입니다.

Cowork 설치 및 시작 방법 (3단계)

1
Claude Desktop 설치

claude.com/download 에서 macOS 또는 Windows 버전을 다운로드합니다. 설치 후 로그인하면 상단에 Chat / Cowork 탭이 표시됩니다.

2
Cowork 탭 선택

앱 상단에서 [Cowork] 탭을 클릭합니다. 일반 Chat과 달리 Claude가 여러 단계의 작업을 자율적으로 실행하는 모드로 전환됩니다.

3
작업 폴더 지정

Claude가 읽고 쓸 로컬 폴더를 허용합니다. 작업할 데이터 파일이 있는 폴더를 지정하세요. 민감한 정보가 있는 폴더는 반드시 제외합니다.

⚠️
Cowork 작업 중 앱을 닫으면 실행이 중단됩니다. 작업이 완료될 때까지 앱을 열어두세요. 진행 상황은 화면에 실시간으로 표시됩니다.

STEP 1 · 준비 — 내 팀 시트 파악하기

ChatGPT에게 요청하기 전, 내 업무와 시트를 먼저 정리합니다

1
현재 시트에서 하는 작업을 목록으로 작성

시트를 어떤 목적으로 사용하는지, 누가 언제 무엇을 입력하는지 구체적으로 적습니다. 모호하게 쓸수록 나중에 ChatGPT 요청도 모호해집니다.

2
시트의 컬럼(항목) 목록 정리

현재 시트에 있는 모든 컬럼 이름을 적습니다. 이것이 웹앱의 입력 항목이 됩니다.

3
누가 어떤 화면을 쓸지 결정

팀원마다 다른 화면이 필요할 수 있습니다. 입력하는 사람과 조회하는 사람이 다르다면 화면을 분리해두면 좋습니다. 이 구분이 웹앱의 탭 메뉴 구조가 됩니다.

4
대시보드에서 보고 싶은 정보 결정

한눈에 보고 싶은 수치나 현황을 미리 생각해두세요. 예: "월별 합계", "항목별 건수", "미완료 항목 표시" 등. 구체적일수록 원하는 결과가 나옵니다.

STEP 2 · ChatGPT에게 코드 요청하기

ChatGPT를 처음 쓰는 분도 따라할 수 있도록 가입부터 코드 복사까지 설명합니다

1

3

  • GPT-4o 모델 사용
  • 복잡한 웹앱 코드 품질 우수
  • 사용량 제한 더 여유로움
  • 처음엔 무료로 시작 권장

④ 생성된 코드 복사하기

ChatGPT가 코드를 응답으로 주면, 아래 방법으로 전체를 복사합니다.

1

응답 중 회색 배경의 코드 블록 찾기 (<!DOCTYPE html>로 시작하는 부분)

2

코드 블록 우측 상단의 "Copy" 버튼 클릭 → 클립보드에 복사됨

3

다음 STEP에서 Claude Cowork 입력창에 붙여넣기 (Ctrl+V / Cmd+V)

코드가 여러 조각으로 나눠서 올 때

GPT가 코드를 2~3개로 나눠서 줄 때가 있습니다. 이때는 같은 대화창에 아래 문장을 그대로 입력하세요.

"위 코드를 전체를 하나의 코드 블록으로 합쳐줘"

⑤ GPT와 대화 이어가며 수정하기

코드를 받은 후 마음에 안 드는 부분이 있으면, 새 창 열지 말고 같은 대화창에 추가로 요청하세요. GPT가 앞의 코드를 기억하고 수정해줍니다.

같은 대화창에서 추가 요청 예시
  • "탭을 3개가 아니라 2개로 줄여줘"
  • "입력 항목에 '비고' 칸 하나 더 추가해줘"
  • "버튼 색상을 파란색으로 바꿔줘"
  • "수정했으면 전체 코드를 다시 하나로 합쳐줘"
⚠️
대화가 너무 길어지면 (20회 이상 주고받으면) GPT가 앞 내용을 잊기 시작합니다. 이때는 새 대화를 시작하고 최종 코드를 붙여넣은 뒤 수정 요청하는 게 더 정확합니다.

STEP 3 · Claude Cowork에서 웹앱 확인하기

ChatGPT가 만든 코드를 Cowork에 붙여넣어 실제 동작하는 웹앱으로 확인합니다

ℹ️
Cowork 설치 및 초기 설정은 앞의 Claude Cowork란? 섹션을 먼저 완료하세요.
1
Claude Cowork 실행 후 Cowork 탭 선택

설치된 Claude Desktop을 실행하고 상단 [Cowork] 탭을 클릭합니다.

2
아래 형식으로 코드를 붙여넣어 요청

메시지와 함께 ChatGPT가 만든 코드를 그대로 붙여넣습니다.

Claude Cowork 입력 형식
아래 HTML 코드를 파일로 저장하고 미리보기로 열어줘.

[ChatGPT가 만든 HTML 코드 전체를 여기에 붙여넣기]

<!DOCTYPE html>
<html lang="ko">
...
</html>
3
미리보기 화면에서 기능 전체 테스트

탭 이동, 데이터 입력·저장, 목록 조회 등 모든 기능을 직접 눌러보며 확인합니다.

4
문제가 있으면 Claude에게 바로 수정 요청

발견한 문제를 일상적인 말로 설명하면 Claude가 코드를 수정합니다. 개발 지식이 없어도 됩니다.

수정 요청 예시
  • "저장 버튼을 눌러도 목록에 데이터가 안 나타나"
  • "날짜 선택 기능이 작동하지 않아"
  • "대시보드 탭에 아무것도 안 보여"
  • "모바일에서 봤을 때 표가 화면을 벗어나"

STEP 4 · 프롬프트 작성 원칙과 수정 방법

좋은 프롬프트가 좋은 결과물을 만듭니다 — 원칙을 알면 수정 횟수가 줄어듭니다

프롬프트 작성 4원칙

Cowork에 처음 지시를 입력할 때 아래 4가지를 모두 포함하면 원하는 결과가 한 번에 나올 확률이 높아집니다.

① 파일·폴더명 명시

파일명, 시트명, 컬럼명을 정확하게 적어주세요.

예시

"sales.csv의 '날짜' 열을 기준으로 집계해줘"

② 원하는 결과 구체화

KPI 항목, 집계 기준, 비교 대상을 명확히 지정하세요.

예시

차트 종류, 색상, 레이아웃을 함께 요청하세요.

0값 처리, 빈 셀 처리, 이상치 기준을 미리 알려주세요.

예시

"0인 값은 집계에서 제외하고, 빈 셀은 '-'로 표시해줘"

STEP 5 · 팀 공유 및 실사용 전환

완성된 웹앱을 팀원과 공유하는 방법 3가지

방법 A · HTML 파일 직접 공유 가장 간단

Claude Cowork가 만든 HTML 파일을 팀 공유 드라이브(Google Drive, SharePoint 등)에 올리고, 팀원이 파일을 받아 브라우저로 실행하는 방식입니다.

1

Claude에게 "현재 HTML 파일을 저장해서 경로 알려줘" 요청 → 파일 다운로드

2

팀 공유 드라이브 폴더에 업로드

3

팀원에게 파일 링크 공유 → 다운로드 후 Chrome으로 실행

⚠️
파일 방식은 각자 PC에 데이터가 저장되므로 팀원 간 데이터가 공유되지 않습니다. 데이터를 팀 전체가 공유해야 한다면 방법 B 또는 C를 사용하세요.

방법 B · GitHub Pages 무료 배포 URL로 공유 가능

인터넷 주소(URL)를 만들어 팀원 누구나 링크로 접속할 수 있게 합니다. 무료이며 설정이 간단합니다.

1

github.com 가입 → 새 저장소 생성

2

HTML 파일을 index.html 이름으로 업로드

3

Settings → Pages → Branch 선택 → Save → URL 자동 생성

4

생성된 URL을 팀원에게 공유

방법 C · Claude에게 배포 요청 Claude가 직접 처리

방법 B의 GitHub Pages 설정을 Claude Cowork가 대신 해줍니다.

자주 묻는 질문 (FAQ)

적용 과정에서 공통으로 나오는 질문들

Q. 코딩을 전혀 몰라도 되나요?

네. 코드를 직접 작성하거나 이해할 필요가 없습니다. 원하는 것을 말로 설명하면 ChatGPT와 Claude가 코드 작성과 수정을 모두 합니다.

Q. ChatGPT 유료 구독이 필요한가요?

무료 버전(GPT-3.5)으로도 시작할 수 있습니다. 다만 여러 화면을 가진 복잡한 웹앱은 GPT-4o 이상에서 코드 품질이 훨씬 좋습니다. 무료로 먼저 시도해보고, 결과가 아쉽다면 업그레이드를 고려하세요.

Q. 데이터가 팀원끼리 공유되지 않아요.

HTML 파일 방식은 각자 PC의 브라우저에 개별 저장됩니다. 팀 전체가 같은 데이터를 봐야 한다면 Claude에게 "Google Sheets를 데이터 저장소로 연동해줘"라고 요청하는 것이 가장 간단한 해결책입니다.

Q. ChatGPT가 만든 코드가 작동하지 않아요.

Claude Cowork 대화창에 "이 코드가 작동하지 않아. 오류를 찾아서 직접 수정해줘"라고 코드와 함께 요청하세요. Claude가 코드를 직접 실행하고 수정합니다. 코드 오류 수정은 ChatGPT보다 Claude가 더 정확합니다.

Q. 완성까지 시간이 얼마나 걸리나요?

준비 메모 작성 10분 → ChatGPT 코드 생성 5분 → Cowork 테스트·수정 30분~1시간이면 기본 웹앱이 완성됩니다. 처음이라면 2~3시간을 여유 있게 잡으세요. 두 번째 팀부터는 훨씬 빠릅니다.

Q. 나중에 기능을 추가하거나 수정할 수 있나요?

언제든지 가능합니다. HTML 파일을 Claude Cowork에 다시 올려서 추가 요청을 하면 됩니다. 수정할 때마다 날짜를 파일명에 붙여 보관하는 것을 권장합니다. (예: 팀명_웹앱_20250401.html)

팀원에게 공유하기 전 아래 항목을 모두 확인하세요

기능 확인

  • 데이터 입력 후 저장이 정상 작동한다
  • 저장한 데이터가 목록에 즉시 반영된다
  • 페이지를 새로고침해도 데이터가 그대로 있다
  • 필수 항목을 비우고 저장 시 안내 메시지가 나온다
  • 검색·필터 기능이 정상 작동한다
  • 삭제 시 확인 팝업이 뜬다

화면·데이터 확인

  • PC 화면에서 레이아웃이 정상적으로 보인다
  • 모바일에서도 잘 보인다
  • 선택 목록의 항목이 실제 업무에 맞게 설정되어 있다
  • 차트·통계가 입력 데이터에 맞게 표시된다
  • 모든 탭이 정상 전환된다

공유 준비

  • 팀원 1~2명에게 사전 테스트를 받았다
  • 공유 방법 (파일 / URL)이 결정되었다
  • 사용 방법 간단 안내문을 준비했다
  • 문의 시 담당자가 누구인지 정해졌다
  • 기존 시트와 병행 사용 기간을 결정했다
